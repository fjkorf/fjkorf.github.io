static structures in a world of change

11.25.2022

I'm proposing a simple change to the way we consider 'variables' in the context of software development.

We learn that 'variables' are containers for fixed values - which is to say that variables are containers for arbitrary finite values that, once set, do not change until set again. This is our mental model for memory, hard-drives, databases, register-machines and countless others. 

x = 7;
From this point on, x is 7.

The reality of the world we are trying to model is that few things stay the same, and most things are in a consistent state of change. This means that at the most fundamental level, our abstraction for computing is at odds with our desired model.

The real world is in constant motion. The volume of water in a river is like the foot traffic of a newly renovated downtown. It is constantly changing. It can be observed, estimated, but never truly known. It is a continuous value. The static value that we think of as the 'variable' is actually a derivative of the value over time. We need a mental model for data that is dynamic by default, and only static in specific situations.

Consider a system where variables are not set, but observed. Multiple observations have the ability to refine the value to provide the best-possible estimation of a true value.

This approach of observe, refine, infer is closer to the way we humans experience the world. It follows that this would serve as the basis for modeling said world. 

    x <= 7;
This does not set the value of x to 7. This asserts that the value of x was observed as 7 at least once. So when you request the value of x, you will only get a 7 if the true value of x is 7 in the context you request it.

If this sounds familiar it's probably because you are familiar with bayesian thinking. We're never defining a constant value, but instead are always updating our priors to get a better understaind of the truth.

An example of this kind of thinking can also be found in implementation of a neural network. No value is constant, and the whole point is to refine complex relationships down to byte-represented values.

I'm proposing a system wherein all ram and hard drive is represented as a continuously transforming neural network. And I'm proposing that this constantly transforming, a-structural configuration will create software that runs faster, is easier to understand, and is more flexible and adaptable to change.

I believe that this is possible with existing technology, and does not require changes to standard computer structure and hardware. Additionally I believe that this concept is orthagonal programming linguistics, and can be applied in any programming language. It is not new technology that will propel us into the next age of computing. It is a return to traditional mathematical reasoning when considering the function of a computer that will be the engine that delivers us to what's next.
